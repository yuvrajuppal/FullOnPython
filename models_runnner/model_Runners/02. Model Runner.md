# How to Run AI Locally with Python

Ollama is a powerful tool for running large language models locally. Because it operates on a Client-Server architecture, setting it up correctly is crucial to avoid connection errors.

### Step 1: Install and Start the Ollama Engine (The Server)

Before Python can talk to Ollama, the core engine must be running in the background.

1. Download the app from [ollama.com](https://ollama.com/) and install it.
    
2. **Crucial:** Ensure Ollama is running.
    
    - On Windows, look for the Llama icon in your system tray (bottom right).
        
    - Alternatively, open a terminal and run `ollama serve`. **Leave this terminal open.**
        
3. Verify the server is active by opening a web browser and navigating to `http://localhost:11434`. If it says "Ollama is running," you are ready.
    
4. Open your terminal and pull a lightweight model suitable for your hardware (e.g., Llama 3.2 1B):
    
    Bash
    
    ```
    ollama pull llama3.2:1b
    ```
    

### Step 2: Install the Python Library

Open a **new** terminal (do not close the one running `ollama serve`) and install the official Python package:

Bash

```
pip install ollama
```

### Step 3: Write the Python Code (The Client)

To prevent IPv4/IPv6 networking conflicts, it is best practice to explicitly point the Python client to `127.0.0.1` rather than relying on `localhost`.

#### Basic Chat (Waits for the full response)

This script sends a message to the model and prints the complete answer once it finishes thinking.



```Python
from ollama import Client
import os

# Clear the terminal for a cleaner output
os.system('cls')

# Explicitly bind to the IPv4 loopback to avoid [::] connection errors
client = Client(host='http://127.0.0.1:11434')

try:
    response = client.chat(model='llama3.2:1b', messages=[
        {
            'role': 'user',
            'content': 'Hello, how are you?',
        },
    ])
    print(response['message']['content'])

except Exception as err:
    print(f"Error connecting to Ollama: {err}")
```

#### Streaming Chat (Prints word-by-word)

For a more interactive, ChatGPT-like experience where the text appears as it is generated, use the `stream=True` parameter.


```Python
from ollama import Client

# Initialize the explicit client
client = Client(host='http://127.0.0.1:11434')

print("Ollama: ", end="", flush=True)

try:
    # The stream=True argument makes the model return chunks of text
    stream = client.chat(
        model='llama3.2:1b',
        messages=[{'role': 'user', 'content': 'Write a short haiku about coding.'}],
        stream=True,
    )

    # Print each chunk as it arrives
    for chunk in stream:
        print(chunk['message']['content'], end='', flush=True)
    
    print() # Add a final newline

except Exception as err:
    print(f"\nError: {err}")
```

### Troubleshooting & Pro Tips

- **The "Two-Terminal" Rule:** If you are running `ollama serve` manually, that terminal is your **Backend Server**. You must open a _second, separate terminal_ to run your Python script (the **Client**). Closing the server terminal kills the engine.
```
(venv) > ollama serve
```

    
- **"Failed to connect / Connection Refused" Error:** The background server isn't running. Check your system tray for the Ollama icon, or run `ollama serve` in a dedicated terminal window.
    
- **"Model not found" Error:** You must download the exact model name referenced in your Python script via the terminal (e.g., `ollama pull llama3.2:1b`) before you can call it.
    
- **"Socket address already in use" Error:** You are trying to run `ollama serve`, but a "ghost" process of Ollama is already running in the background. Open Task Manager, force-quit any tasks named "Ollama", and try again.
    
- **Performance Tip for Budget Hardware:** If you have limited RAM (e.g., 6 GB or less), close your web browser and any Electron-based apps (like VS Code or Discord) before running your script. Python and the LLM will compete for memory, and freeing it up prevents severe lag.
