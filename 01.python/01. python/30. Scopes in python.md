Python, **scope** determines which parts of your program can "see" or access a specific variable. Think of it like a set of security clearances: some variables are available everywhere, while others are locked inside specific rooms (functions).

Python follows the **LEGB rule** to decide which variable to use. It searches in this specific order:

### The LEGB Rule

1. **L**ocal: Variables defined inside a function.
    
2. **E**nclosing: Variables in the local scope of nested (inner) functions.
    
3. **G**lobal: Variables defined at the top level of a script or module.
    
4. **B**uilt-in: Reserved keywords like `print`, `len`, or `range`.
    

---

### Examples of Scope

#### 1. Local vs. Global Scope

A variable created inside a function belongs to the **Local** scope of that function and cannot be used outside of it.



```Python
x = "I am Global" # Global scope

def my_function():
    y = "I am Local" # Local scope
    print(y)
    print(x) # Functions can "see" global variables

my_function()
# print(y)  <-- This would cause a NameError!
```

#### 2. Enclosing (Non-local) Scope

This happens when you have a function inside a function.

```Python
def outer_function():
    name = "Alice" # Enclosing scope for inner_function
    
    def inner_function():
        print(name) # Accesses the variable from the outer function
        
    inner_function()

outer_function()
```

---

### Modifying Variables Across Scopes

By default, you can _read_ a global variable inside a function, but you cannot _change_ it unless you use specific keywords.

#### The `global` Keyword

If you need to update a global variable from inside a function:

```Python
counter = 0

def increment():
    global counter # Tells Python to use the top-level variable
    counter += 1

increment()
print(counter) # Output: 1
```

#### The `nonlocal` Keyword

Used inside nested functions to modify a variable in the outer (enclosing) function:

Python

```
def outer():
    count = 0
    def inner():
        nonlocal count
        count += 1
    inner()
    return count
```

---

### Why does scope matter?

- **Avoids Name Collisions:** You can use the variable name `i` in ten different functions without them interfering with each other.
    
- **Memory Efficiency:** Local variables are usually deleted from memory once the function finishes running.
    
- **Predictability:** It prevents accidental changes to global data that might break other parts of your program.
