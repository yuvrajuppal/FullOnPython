Python 3.10 introduced the `match` statement, often called **Structural Pattern Matching**. If you’ve used languages like C++ or Java, it might look like a `switch` statement, but it’s actually much more powerful because it can "deconstruct" objects to see what's inside them.

### How it Works

The basic syntax uses the `match` keyword followed by `case` blocks. Python compares the expression to each pattern from top to bottom.

### Basic Example: Replacing If-Else

Instead of a long chain of `if/elif/else` for simple values, you can use `match`:

Python

```
status = 404

match status:
    case 200:
        print("Success!")
    case 400 | 404:  # The | operator acts as "OR"
        print("Not found or Bad Request")
    case 500:
        print("Server Error")
    case _:  # The underscore is a wildcard (like 'else')
        print("Unknown status")
```

---

### Advanced Example: Pattern Deconstruction

The real "magic" happens when you use it to unpack data structures like lists or dictionaries.

Python

```
command = ["move", "10", "20"]

match command:
    case ["quit"]:
        print("Goodbye!")
    case ["move", x, y]:
        print(f"Moving to coordinates: {x}, {y}")
    case ["jump", *coords]: # Use * to capture multiple values
        print(f"Jumping to: {coords}")
    case _:
        print("Command not recognized")
```

### Key Features to Remember

- **The Wildcard (`_`):** This matches anything. It is always placed at the end to catch any cases you didn't specifically define.
    
- **Combining Cases:** Use the pipe symbol `|` to match multiple patterns in one case block.
    
- **Guards:** You can add an `if` statement to a case for extra filtering (e.g., `case x if x > 0:`).
    
- **No "Fall-through":** Unlike some languages, Python executes only the _first_ match it finds and then exits the block. You don't need `break` statements.
