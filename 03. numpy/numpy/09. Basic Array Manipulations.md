### 1. Mathematical Operations on Arrays

In NumPy, basic mathematical operations are **element-wise**. This means if you add two arrays together, it adds the first element of array A to the first element of array B, the second to the second, and so on.

You can use standard Python operators (`+`, `-`, `*`, `/`, `**`) directly on the arrays.

**Example:**

```Python
import numpy as np

# Create two 1D arrays
array_a = np.array([10, 20, 30])
array_b = np.array([1, 2, 3])

# Element-wise operations
addition = array_a + array_b       # [11, 22, 33]
subtraction = array_a - array_b    # [9, 18, 27]
multiplication = array_a * array_b # [10, 40, 90]
division = array_a / array_b       # [10.0, 10.0, 10.0]

print("Multiplication Result:", multiplication)
```

### 2. Universal Functions (ufuncs)

A Universal Function (or **ufunc**) is a function that operates on arrays in an element-by-element fashion.

While the basic operators (`+`, `-`, etc.) are technically ufuncs behind the scenes, NumPy provides a massive library of advanced mathematical ufuncs (like trigonometry, exponents, and statistical functions). The primary benefit of ufuncs is **vectorization**—they are pushed down to compiled C code, making them infinitely faster than looping through an array in standard Python.

**Example:**

```Python
# Create an array
data = np.array([1, 4, 9, 16])

# Calculate the square root of every element instantly
roots = np.sqrt(data)
print("Square Roots:", roots) # Output: [1. 2. 3. 4.]

# Calculate the exponential (e^x) of every element
exponents = np.exp(data)
print("Exponentials:", exponents)
```

### 3. Broadcasting Rules

Broadcasting is easily the most powerful (and sometimes the most confusing) feature in NumPy. It is a set of rules that allows you to perform mathematical operations on arrays of **different shapes**.

Instead of throwing an error because the arrays don't perfectly match, NumPy will automatically "stretch" or "broadcast" the smaller array across the larger one so that they have compatible shapes.

**The Strict Rules of Broadcasting:** NumPy compares the dimensions of the two arrays starting from the rightmost dimension and working left. Two dimensions are compatible for math if:

1. **They are equal**, OR
    
2. **One of them is exactly 1.** (If a dimension is 1, NumPy will duplicate the data along that dimension to match the other array).
    

If neither of these conditions is met, NumPy throws a `ValueError`.

**Example:** Adding a 1D array to a 2D matrix.

```Python
# A 2x3 matrix (2 rows, 3 columns)
matrix = np.array([[1, 2, 3],
                   [4, 5, 6]])

# A 1D array with 3 elements (conceptually 1 row, 3 columns)
row_to_add = np.array([10, 20, 30])

# Broadcasting in action!
result = matrix + row_to_add

print(result)
# Output:
# [[11 22 33]
#  [14 25 36]]
```

**What happened here?** The `row_to_add` had a shape of `(3,)`. The matrix had a shape of `(2, 3)`. Because the rightmost dimensions matched (both are 3), NumPy virtually duplicated `row_to_add` so it acted like a `(2, 3)` matrix: `[[10, 20, 30], [10, 20, 30]]`. Then, it added them together seamlessly!
