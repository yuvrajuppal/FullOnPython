### **1. Fancy Indexing**

**Fancy indexing** is a term adopted by NumPy to describe indexing an array using arrays (or lists) of indices rather than scalars or slices. This allows you to access or modify multiple specific elements at once, in any order you choose.

**How it works:** Instead of `arr[0:5]` (slicing), you pass a list of the exact indices you want: `arr[]`.

### **2. Boolean Indexing (Masking)**

**Boolean indexing** involves using a boolean array (an array of `True` and `False` values) to filter or select data from another array. The resulting array will only contain the elements where the boolean array was `True`.

This is incredibly useful for filtering datasets based on specific conditions.

**How it works:** You create a "mask" by applying a logical condition to an array (e.g., `arr > 5`). You then pass that mask into the array's brackets: `arr[mask]`.

**Example:**

Python

```Python
import numpy as np

data = np.array([15, -3, 22, 8, -9, 45, 0])

# 1. Create a boolean mask where elements are greater than 0
positive_mask = data > 0
print("Boolean Mask:", positive_mask)
# Output: [ True False  True  True False  True False]

# 2. Apply the mask to filter the array
positive_numbers = data[positive_mask]
print("Filtered Array:", positive_numbers)
# Output: [15 22  8 45]

# 3. Doing it in one line (most common practice)
even_numbers = data[data % 2 == 0]
print("Even Numbers:", even_numbers)
# Output: [ 8  0]

# 4. Modifying data based on a condition
# Set all negative numbers to 0
data[data < 0] = 0
print("Modified Array:", data)
# Output: [15  0 22  8  0 45  0]
```

### **3. Working with Structured Arrays and Record Arrays**

Standard NumPy arrays can only hold a single data type (e.g., all integers or all floats). **Structured arrays** allow you to define compound data types so that a single array can hold heterogeneous data.

Conceptually, you can think of a structured array exactly like a table in a relational database (like SQL) where each column has a specific data type (string, integer, float) and each element in the array represents a single row or record.

**Record arrays** (`np.recarray`) are a subclass of structured arrays. They hold the exact same data, but they allow you to access columns as _attributes_ (e.g., `data.age`) rather than as dictionary keys (e.g., `data['age']`).

**Example:**

Python

```Python
import numpy as np

# 1. Define the compound data type (similar to a SQL schema)
# 'U10' = Unicode string of max length 10
# 'i4' = 32-bit integer
# 'f4' = 32-bit float
schema = [('name', 'U10'), ('age', 'i4'), ('salary', 'f4')]

# 2. Create the Structured Array
employees = np.array([
    ('Alice', 28, 75000.0),
    ('Bob', 34, 82000.5),
    ('Charlie', 22, 55000.0)
], dtype=schema)

print("Structured Array:\n", employees)

# Accessing data like a dictionary key (Structured Array style)
print("\nAll Names:", employees['name'])
# Output: ['Alice' 'Bob' 'Charlie']

print("Alice's Age:", employees['age'])
# Output: 28

# 3. Using Record Arrays for attribute access
# Convert the structured array into a record array
rec_employees = employees.view(np.recarray)

# Now you can access columns as attributes
print("\nAll Salaries (via attribute):", rec_employees.salary)
# Output: [75000.  82000.5 55000. ]

# Combining Concepts: Boolean indexing on a record array
# Find names of employees making over 60k
high_earners = rec_employees[rec_employees.salary > 60000].name
print("High Earners:", high_earners)
# Output: ['Alice' 'Bob']
```
